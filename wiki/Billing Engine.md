# Billing Engine

Billing is a **document-generating engine**, not a passive "preparation signal" — the single biggest scope decision in the project (ADR-001 V2, 2026-06-30, which supersedes older wording — see [[Docs Precedence]]). Visible to Admin/Coordinator only; **entirely hidden from Trainers**.

## What the app bills (TVI-scope)

The **TVI bills** three document types, generated by populating the school's `.docx` template (per-school header, tenant signatories, TESDA PO addressee — from `tenant_settings`):

1. **TSF / Allowance** (TSF + New Normal + Insurance for CFSP)
2. **Training Cost**
3. **Entrepreneurship** — ₱800 × scholars *without* the `entrepreneurship_completed` flag

The **Assessment Center bills the Assessment Fee** — out of app scope. The app also never submits anything to TESDA.

## Amounts come from snapshots

The TESDA Schedule of Cost (Circular 015 s.2026) is a seeded reference table (`scholarship_cost_schedule`) keyed by program + qualification; the **batch snapshots its cost row at creation** (BB1). Rates: TSF ₱160/day, New Normal ₱1,000 (CFSP), Insurance ₱100.80, Entrepreneurship ₱800. Worked CFSP example (360 hrs → 45 days): TSF = 45 × ₱160 = ₱7,200/scholar; cohort example totals **₱137,000** — the acceptance benchmark.

## Release schedules (regulator-fixed)

**TSF** (rules 7.2.2/7.2.3 — trigger is per-scholar attendance %):

| Calendar duration | Tranches |
| --- | --- |
| < 2 months | 50% @ ≥20% · 50% @ ≥80% |
| ≥ 2 months | 20% @ ≥20% · 40% @ ≥50% · 40% @ ≥80% |

The **absence deduction** (`absences × ₱160`) applies to the **final tranche in both schedules** (FF2).

**Training Cost** (rules 5.1.1/5.1.2 — boundary is **60 *training* days**, distinct from TSF's 2-*month* boundary; a batch runs both schedules in parallel): < 60 days → full payment at completion; > 60 days → 50% @ ≥50% duration · 50% at end. Amount = `training_cost_snapshot × eligible_scholars`.

**Entrepreneurship:** single payment on delivery of the 3-day block; no tranche rule. The `entrepreneurship_completed` flag is a dual-training guard with three effects: TSF −₱480, duration −3 days, excluded from the ₱800 billing (Flag-3x).

## Readiness, generation log, and the "no ledger" rule

- **Billing-ready** = threshold reached **AND** all that tranche's supporting documents verified ([[Document Checklist And Evidence]]).
- Each generation appends a **versioned, append-only `billing_records` snapshot** — never overwritten. Attendance stays editable post-billing; correct → re-generate → new version (Y-hybrid).
- **No envelope ledger / no over-billing guard** — the TESDA Provincial Office reconciles (NoLedger). `approved_slots` on the batch is the billable-pax cap ([[RQM And NTP Authorization]]).
- Output formatting: scholar rows **alphabetized by last name + numbered**; total rendered **in words**.

## Where it lives in code (planned)

Pure domain logic in `lib/domain/billing/*` (eligibility, tranches, amounts — deterministic, unit-testable with fixed as-of dates); persistence in `lib/data/billing.ts`; `.docx` templating in `lib/billing/`. Built in Phase 4B of the [[Implementation Roadmap]].
